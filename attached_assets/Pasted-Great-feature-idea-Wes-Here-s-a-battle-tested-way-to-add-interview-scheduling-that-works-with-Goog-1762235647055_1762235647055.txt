Great feature idea, Wes. Here’s a battle-tested way to add interview scheduling that works with Google Calendar, Outlook/Microsoft 365 (incl. Teams), and Zoom—cleanly in your React + Express stack on Replit.

# 1) Pick your integration approach

**A. Build direct, first-party integrations (full control, more work)**

* **Availability**

  * Google: use **FreeBusy** to read slots. ([Google for Developers][1])
  * Microsoft 365/Outlook: use **findMeetingTimes** or **getSchedule** via Graph. ([Microsoft Learn][2])
* **Create the event + video link**

  * Google: **events.insert** with `conferenceData.createRequest` (Meet). (Remember `conferenceDataVersion=1`.) ([Google for Developers][3])
  * Microsoft/Teams: **/onlineMeetings** to create a Teams link, or create a calendar **event** and attach Teams. ([Microsoft Learn][4])
  * Zoom: create a meeting via Zoom OAuth app + granular scopes. ([Zoom][5])
* **Stay in sync**

  * Google: enable **push notifications** (`events.watch`) to avoid polling. ([Google for Developers][6])
  * Microsoft: **change notifications** (webhooks) for calendar events. ([Microsoft Learn][7])

**B. Use a unified calendar layer (fastest to ship, $$)**

* **Cronofy** (real-time availability & multi-attendee logic). ([Cronofy][8])
* **Nylas** (unified Calendar API + embeddable scheduler). ([Nylas][9])
* **Cal.com** (open-source scheduler you can embed or self-host; has APIs and Teams/Zoom apps). ([Cal.com][10])
  (Heads-up: Cronofy pricing is enterprise-leaning.) ([zeeg.me][11])

**Recommendation:**

* If you want white-label control inside Sebenza Hub and can invest dev time: go **direct**.
* If you want a quick MVP with rich UX and routing rules: embed **Cal.com** now, then phase into direct APIs later. ([Cal.com][10])

# 2) Core user flows to implement

1. **Interviewer availability**

* HR sets working hours, buffers, locations (On-site/Remote/Hybrid), meeting lengths, and interview “pools” (round-robin).
* Sync each interviewer’s connected calendar(s); persist OAuth tokens + provider in DB.

2. **Candidate booking**

* Candidate picks: time zone (default **Africa/Johannesburg**), mode (Zoom/Teams/Meet), slot.
* Server validates slot via provider free/busy right before booking to prevent double-books.

3. **Event creation & invites**

* Create provider event (add organizer + candidate + panelists).
* If video:

  * Google → `conferenceData.createRequest` (Meet). ([Google for Developers][12])
  * Teams → `POST /onlineMeetings` or event with Teams provider. ([Microsoft Learn][4])
  * Zoom → `POST /meetings` with OAuth app + scopes. ([Zoom][5])
* Send email/WhatsApp confirmations + .ics fallback.

4. **Reschedule/Cancel**

* Signed link in emails; PATCH/DELETE the event at the provider and notify all parties.

5. **Sync & webhooks**

* Subscribe to **Google** events.watch and **Graph** change notifications so external edits reflect in Sebenza Hub instantly. ([Google for Developers][13])

# 3) Minimal technical design (React + Express/TS)

**Frontend**

* Availability UI: calendar + slot list; timezone selector; video provider toggle; interviewer pool selector.
* Candidate form: name, email, phone, role, location, notes; consent checkbox.

**Backend**

* **Auth/OAuth**

  * Google OAuth2 (offline access, store refresh token).
  * Microsoft identity platform OAuth2 (delegated permissions).
  * Zoom OAuth app with granular scopes.
* **Endpoints (examples)**

  * `GET /availability?poolId=...&from=...&to=...`
  * `POST /book` → validates slot, creates provider event + conferencing, sends notifications.
  * Webhooks:

    * `/webhooks/google` (verify channel headers; maintain sync tokens). ([Google for Developers][6])
    * `/webhooks/graph` (subscription validation & notifications). ([Microsoft Learn][7])
    * `/webhooks/zoom` for meeting.created/updated (optional). ([Zoom][5])
* **DB tables (sketch)**

  * `connected_accounts`: userId, provider, email, scopes, refreshToken, expiresAt
  * `interview_pools`: id, name, routing (roundRobin|allAvailable|bestFit), bufferMins
  * `pool_members`: poolId, interviewerUserId, weight
  * `interviews`: id, candidateId, providerEventId, provider, meetingJoinUrl, status
  * `holds`: userId, start, end, reason (for blocks/buffers)

**Availability logic**

* Google: POST `/calendar/v3/freeBusy` with merged calendars; interpret busy windows. ([Google for Developers][1])
* Microsoft: call **findMeetingTimes** for multi-attendee suggestions, or **getSchedule** for raw busy times. ([Microsoft Learn][2])

**Event creation (examples)**

* Google: `events.insert` with `conferenceData` (Meet), set `conferenceDataVersion=1`. ([Google for Developers][3])
* Teams: `POST /onlineMeetings` then include join info in calendar event or send directly. ([Microsoft Learn][4])
* Zoom: `POST /meetings` then add link in event description/invite. ([Zoom][5])

---

# 4) Scheduling features recruiters will love (and how)

* **Round-robin & load balancing**: route bookings across a pool; persist counters/weights.
* **Panel interviews**: availability = intersection of *N* calendars; use Graph **findMeetingTimes** to reduce search. ([Microsoft Learn][2])
* **Buffers, min notice, working hours**: enforce in your slot generator before provider calls.
* **On-site vs Remote**: if on-site, include location map & parking notes; if remote, attach video link.
* **Time-zone smartness**: store recruiter TZ; render candidate TZ; always persist UTC.
* **No-show handling**: one-click reschedule; auto-send reminder emails/SMS/WhatsApp (Twilio/Meta WhatsApp Cloud API).
* **.ics fallback**: always attach an iCalendar file for external calendars.

---

# 5) Security & compliance checklist

* Least-privilege scopes; show users exactly what you’re requesting.
* Encrypt refresh tokens at rest; rotate webhook secrets.
* Handle Google/Graph webhook validations; maintain sync tokens to avoid missing changes. ([Google for Developers][6])
* GDPR/POPIA: explicit consent, data-retention windows, delete on disconnect.

# 6) Fastest MVP path (2 options)

**Option 1 — Embed & ship this week**

* Stand up **Cal.com** (hosted or self-hosted), style it to Sebenza Hub, embed booking per role/team (supports Teams/Zoom/Meet via its apps). Then log bookings back to your DB via webhooks. ([Cal.com][10])

**Option 2 — Direct integrations (gradual)**

1. Google only: free/busy → events.insert(+Meet) → push notifications. ([Google for Developers][1])
2. Add Microsoft Graph (Outlook/Teams). ([Microsoft Learn][2])
3. Add Zoom creation & webhooks. ([Zoom][5])
